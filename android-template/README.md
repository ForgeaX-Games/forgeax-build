# ForgeaX Android Studio Project (exported)

This directory is a standard Android Studio project generated by ForgeaX Studio
("Package Game → Android"). The game runs inside a WebView, served from local
assets over a secure `https://appassets.androidplatform.net/` origin (via
`WebViewAssetLoader`) so WebGPU / Secure-Context-gated APIs work without any
network permission.

## Open & build

1. Open this folder in **Android Studio** (Giraffe or newer).
2. Let Gradle sync (the Gradle wrapper is bundled — works offline once the
   distribution is cached).
3. Run on a device/emulator, or `Build → Generate Signed Bundle / APK` for
   release.

## Integrate third-party SDKs

Add SDKs (payment / ads / login) in `app/build.gradle.kts` `dependencies {}` and
wire them in `app/src/main/java/com/forgeax/player/MainActivity.kt`. To call
native code from the game JS, add a `@JavascriptInterface` bridge to the WebView.

## Requirements / caveats

- **System WebView**: WebGPU on Android WebView is recent. Test devices need an
  up-to-date **Android System WebView / Chrome**; older WebViews may fall back to
  WebGL (if the engine supports it) or show a blank screen.
- **Icon**: `app/src/main/res/drawable/app_icon.*` is the launcher icon. The
  exporter writes a single image; for crisp multi-density / adaptive icons,
  replace it via Android Studio's Image Asset wizard.
- **Signing**: the project ships with debug config only; configure your release
  keystore in Android Studio.

## Layout

```
app/src/main/
  AndroidManifest.xml
  assets/public/            # the web game (index.html, *.js, *.wasm, assets)
  res/values/strings.xml    # app_name
  res/drawable/app_icon.*   # launcher icon
  java/com/forgeax/player/
    MainActivity.kt         # WebView host + asset loader wiring
    ForgeaxPathHandler.kt   # fixes wasm/js MIME for instantiateStreaming
```
